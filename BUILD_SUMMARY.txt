================================================================================
                      SMART CLASSIFICATION RULES
================================================================================

Priority order for automatic transaction classification:

1. Note contains "da dividere"
   → persona = "Da Dividere"

2. Note contains "addebitare a luisa" OR "per luisa"
   → persona = "Moglie"

3. Note contains "- virgilio" OR "– virgilio"
   → persona = "Virgilio"

4. Note contains "luisa" (alone, not with virgilio)
   → persona = "Moglie"

5. Note contains "moglie"
   → persona = "Moglie"

6. Note contains "posti"
   → categoria = "Spese Lavoro", persona = "Virgilio"

7. Note contains "santa severa"
   → categoria = "Casa Santa Severa"

8. Note contains "settimana bianca"
   → categoria = "Settimana Bianca", persona = "Da Dividere"

9. Note contains "compleanno gigi" + "dividere"
   → categoria = "Viaggio Compleanno Gigi", persona = "Da Dividere"

10. Description contains "garage"
    → categoria = "Box Auto/Parcheggio"

11. Description contains "commissioni" OR "costo bonifico"
    → categoria = "Commissioni Bancarie", persona = "Virgilio"

12. fonte = "Carta Gigi"
    → persona = "Figlio"

User can override any classification via inline editing.

                          QUICK START

TO START DEVELOPING:

  1. Open terminal in project directory
  2. Run: npm run dev
  3. Open: http://localhost:3000
  4. Navigate using sidebar menu
  5. Go to /upload to import transaction data
  6. Use /transactions to view and edit
  7. Check Dashboard for analytics

TO IMPORT DATA:

  1. Prepare Excel (.xlsx) or CSV (.csv) file with columns:
     data, descrizione, dettaglio, importo, macroCategoria, persona, note, fonte

  2. Navigate to Upload page
  3. Drag & drop file (or click to select)
  4. Choose bank source
  5. Review preview
  6. Click "Importa Transazioni"

TO ADD SPENDING LIMITS:

  1. Go to Settings page
  2. Select category and enter monthly limit
  3. Click "Aggiungi"
  4. Toggle active/inactive as needed
  5. Limits appear as red alerts on Dashboard when exceeded

================================================================================
                        TROUBLESHOOTING GUIDE
================================================================================

ISSUE: "Module not found" error
FIX: Run "npm install" to ensure all dependencies installed

ISSUE: Build fails with TypeScript errors
FIX: Check error line number and fix type issues, rebuild

ISSUE: Dev server won't start
FIX: Check port 3000 isn't in use, kill process: lsof -i :3000

ISSUE: Data not persisting
FIX: Browser may have blocked IndexedDB. Check privacy settings.

ISSUE: Import file fails
FIX: Verify file format (Excel or CSV), check column names match exactly

ISSUE: Charts not showing
FIX: Ensure you have transactions in the date range, refresh page

ISSUE: Sidebar icons not showing
FIX: Lucide React icons library should auto-load, check network tab
